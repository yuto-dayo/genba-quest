# PR-05.5 — FE: 立替詳細モーダル(ExpenseDetailModal)

> **Addendum (2026-05-16)**: PR-05 までマージ済 or stack 済を前提。立替カルーセル(`MemberCarousel` mode="expense")のカードタップ / 全員を見る が console.log で止まっている UX 穴を埋める。`OwnRewardModal.tsx` の構造をコピペ流用し、`fetchMemberReimbursementBalance(memberId, month)` をデータソースに置換。

## Goal
立替カルーセルのカードタップ(自分 / 他人どちらも)で開く詳細モーダルを実装。職人が「自分の立替¥45K の内訳は?」「タケさんの立替が今いくら積み上がってる?」を3秒で確認できるようにする。同時に「全員を見る」カードからチーム全員の立替を一覧する modal を提供する。

## Why
MVP「立替経費可視化」の最後のピース。報酬フロー(PR-04/05)は完結しているが、立替フローはヒーロー数字までしか見えず、内訳・直近項目・精算状態が未到達。職人が「自分の立替どこから来てる?」に答えがないと信頼が落ちる。

## Component spec

### `<ExpenseDetailModal memberId month onClose onExpenseAdded?>`
- Props:
  - `memberId: string` — 表示対象(自分も他人もこの ID で fetch)
  - `month: string`
  - `selfMemberId: string | null` — 自分判定用
  - `onClose: () => void`
  - `onExpenseAdded?: () => Promise<void> | void` — `[+ 経費を追加]` 完了時に呼ばれる(親で carousel refresh)
- 状態:
  - loading / empty / error
  - data: `MemberReimbursementBalance`(PR-02 で定義済)
- 構造: `OwnRewardModal.tsx` をコピペ。scrim, modal shell, header, body, footer は同じ。AbortController + getErrorMessage 流用。
- 表示要素(per `feedback_money_design_principles.md` 数字主役):
  - ヘッダ: `${month} の立替`
  - 主役メトリクス: 合計立替額(`total_advanced`)を `display-small` で
  - サブメトリクス: `未精算 ¥X / 精算済 ¥Y`
  - 状態別小計テーブル(4 行): 申請待ち / 申請済 / 承認済 / 振込済 ←→ `by_status` の値
  - 直近の経費(最大5件): 日付 / カテゴリ / 金額 / 状態チップ
  - 自分のモーダルのみ footer 左に `[+ 経費を追加]`
- a11y: `role="dialog"`、`aria-modal="true"`、`aria-labelledby`

### `<TeamExpenseSummaryModal month onClose onExpenseClicked>`
- `fetchMemberReimbursementsSummary(month)` を fetch
- メンバー一覧を縦リスト表示(nickname / 合計立替 / 未精算 / 状態)
- 各行タップで `onExpenseClicked(memberId)` を呼んで `ExpenseDetailModal` に切替
- TeamSummaryModal と同じ構造で OK(reward / expense で抽象化したいなら別 PR でリファクタ)
- ローディング / エラー / 空(全員 0 件)状態

## Status chip mapping(`feedback_plain_language_ui.md` 準拠)

| `reimbursement_status` | UI ラベル |
|---|---|
| `unsubmitted` | 申請待ち |
| `submitted` | 申請済 |
| `approved` | 承認済 |
| `reimbursed` | 振込済 |

色は `--money-status-*` を使う(pending = warning / completed = success など)。色だけで意味を伝えず必ずテキスト併記。

## Category アイコン(任意、無くてもOK)

`recent_items[].category` 値が固定enum なら絵文字でアフォーダンス足してもOK:
- 駐車 → 🅿️ / 材料 → 🛒 / ガソリン → ⛽ / 食事 → 🍱 / 工具 → 🔧 / その他 → 📄
これは無くても accept、UI が静かなら適用しない。

## Edge cases

- `recent_items.length === 0`(直近項目なし、合計は 0 でない): 「直近の経費はありません」inline
- `total_advanced === 0`: モーダル全体が「立替なし」状態。直近項目もテーブルも非表示、`[+ 経費を追加]`(自分のみ)だけ表示
- 自分かどうか不明(`selfMemberId === null`): `[+ 経費を追加]` 非表示、`[閉じる]` のみ
- ネットワークエラー: 既存パターン(error panel + retry)
- 月切替中の race: AbortController で前リクエストキャンセル
- `[+ 経費を追加]` 押下後のリフレッシュ: 経費が追加されたら ExpenseDetailModal 自身も再 fetch し、`onExpenseAdded` で親 hero も更新

## Forbidden

- 領収書本文 / 店舗名 / 振込先など PII を表示(BE で返さない設計、念のため UI 側でも参照しない)
- 自分モーダルと他人モーダルを別コンポーネントに分ける(同一 + props で分岐)
- typed confirmation
- `M3` トークン値変更
- ハードコード金額 / 色
