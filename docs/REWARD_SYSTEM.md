# 報酬分配システム設計書

## 概要

職人の技術力に応じて現場利益を公平に分配するシステム。
**「技術を上げれば報酬が上がる」** という明快なインセンティブ設計。

---

## 基本方針

| 項目 | 方針 |
|------|------|
| 報酬決定要素 | **Tスコア（技術力）× 稼働日数** のみ |
| 行動評価 | 報酬に含めない（別途フィードバック運用） |
| 報酬格差 | 同じ稼働日数で最大3〜5倍に収める |
| 透明性 | 計算ロジックを全員に公開 |

## 評価プロセス

### 3視点評価

| 視点 | 評価者 | 役割 |
|------|--------|------|
| 自己評価 | 本人 | 達成したと思う項目を申告 |
| 360°レビュー | チームメンバー | 他メンバーから見た評価 |
| 熟練者レビュー | 最も経験豊富な人 | 最終判断・根拠確認 |

### スター認定ルール

1. **2票以上一致** → 確定
   - 3人中2人以上が「達成」と判断した項目を認定

2. **票が割れた場合**
   - 熟練者が明示根拠つきで主張 → 熟練者を採用
   - 根拠不十分 → 不認定（安全側に倒す）

3. **原則**
   - 推測での加点はしない
   - 実際に行った実例のみで評価

## 報酬計算

### 計算式

```
分配対象利益 = 現場純利益 × (1 - 会社取り分率)

T_norm = (Tスコア ÷ 170) × 100      # 0〜100に正規化
T_boost = (T_norm ÷ 100)^p × 100    # 非線形ブースト

effort = 稼働日数 × T_boost
ratio = effort ÷ Σeffort（全員の合計）

支給額 = 分配対象利益 × ratio
```

### パラメータ

| パラメータ | デフォルト | 説明 |
|-----------|-----------|------|
| company_rate | 0.00 | 会社取り分率（0〜1）→ 全額職人に分配 |
| p（ブースト指数） | 1.5 | >1で差が拡大、<1で差が縮小 |
| min_payment | 0 | 最低保証額 |
| max_payment | なし | 上限額（任意） |

### ブースト指数(p)の効果

| p値 | 効果 | T_norm=50のT_boost |
|-----|------|-------------------|
| 1.0 | 線形（差なし） | 50 |
| 1.5 | 中程度の差 | 35 |
| 2.0 | 標準（推奨） | 25 |
| 2.5 | 大きな差 | 18 |

p=2.0の場合、技術力の差が報酬に2乗で反映される。
例: T_norm=50 → T_boost=25、T_norm=100 → T_boost=100（4倍差）

### 報酬格差の設計

**目標: 同じ稼働日数で最大3〜5倍**

| レベル | Tスコア | T_norm | T_boost(p=2) | 報酬比 |
|--------|---------|--------|--------------|--------|
| 見習い | 34点 | 20% | 4 | 1.0x |
| 中堅 | 85点 | 50% | 25 | 6.3x |
| 熟練 | 136点 | 80% | 64 | 16x |
| マスター | 170点 | 100% | 100 | 25x |

※見習いと熟練の差が大きすぎる場合は p=1.5 に調整

**p=1.5の場合**

| レベル | T_boost | 報酬比 |
|--------|---------|--------|
| 見習い(20%) | 9 | 1.0x |
| 中堅(50%) | 35 | 3.9x |
| 熟練(80%) | 72 | 8.0x |
| マスター(100%) | 100 | 11x |

→ **p=1.5〜1.8が3〜5倍の目標に近い**

## 計算例

### 条件
- 現場純利益: 100万円
- 会社取り分率: 10%
- 分配対象: 90万円
- ブースト指数: p=1.5

### メンバー

| 名前 | 稼働日数 | Tスコア | T_norm | T_boost |
|------|---------|---------|--------|---------|
| Aさん | 5日 | 136点 | 80% | 72 |
| Bさん | 5日 | 85点 | 50% | 35 |
| Cさん | 3日 | 51点 | 30% | 16 |

### 計算

| 名前 | effort | ratio | 支給額 |
|------|--------|-------|--------|
| Aさん | 5×72=360 | 58.4% | 525,600円 |
| Bさん | 5×35=175 | 28.4% | 255,600円 |
| Cさん | 3×16=48 | 7.8% | 70,200円 |
| **合計** | 583 | 100% | 900,000円 |

**報酬比**
- Aさん : Bさん = 2.1倍
- Aさん : Cさん = 7.5倍（稼働日数差込み）
- 同じ稼働日数ならAさん : Cさん = 4.5倍

## アプリ実装

### データモデル

```typescript
// 職人のTスコア
interface WorkerSkill {
  id: string;
  worker_id: string;
  skill_item_id: string;     // スキル項目ID
  achieved: boolean;         // 達成済みか
  achieved_at?: string;      // 達成日
  evidence?: string;         // 根拠（任意）
}

// スキル項目マスタ
interface SkillItem {
  id: string;
  category: 'pate' | 'cross';  // パテ or クロス
  name: string;
  name_en: string;
  points: number;              // 配点
  description?: string;
  sort_order: number;
}

// Tスコア評価セッション
interface TScoreEvaluation {
  id: string;
  worker_id: string;
  evaluated_at: string;
  self_score: number;         // 自己評価点
  peer_score: number;         // 360°平均点
  master_score: number;       // 熟練者評価点
  final_score: number;        // 確定Tスコア
  evaluators: string[];       // 評価者ID
}

// 報酬計算
interface RewardCalculation {
  id: string;
  site_id: string;
  calculated_at: string;
  profit: number;             // 現場純利益
  company_rate: number;       // 会社取り分率
  distributable: number;      // 分配対象額
  boost_p: number;            // ブースト指数
  results: RewardResult[];
}

interface RewardResult {
  worker_id: string;
  days: number;               // 稼働日数
  t_score: number;            // Tスコア
  t_norm: number;             // 正規化スコア
  t_boost: number;            // ブースト後スコア
  effort: number;             // effort値
  ratio: number;              // 分配比率
  amount: number;             // 支給額
}
```

### API設計

```
# Tスコア関連
GET  /api/v1/skills                    # スキル項目一覧
GET  /api/v1/workers/:id/skills        # 職人のスキル状況
POST /api/v1/workers/:id/skills/:itemId/achieve  # スキル達成登録
POST /api/v1/evaluations               # 評価セッション作成
GET  /api/v1/evaluations/:id           # 評価結果取得

# 報酬計算関連
POST /api/v1/sites/:id/calculate-reward  # 報酬計算実行
GET  /api/v1/sites/:id/rewards           # 報酬計算履歴
```

### UI設計

1. **スキル管理画面**（Settings内）
   - 職人ごとのスキル達成状況
   - 現在のTスコア表示
   - スキルツリー風の可視化

2. **評価フロー**（定期実施）
   - 自己評価入力
   - 360°レビュー入力
   - 熟練者レビュー入力
   - 結果確認・確定

3. **報酬計算**（Sites詳細 or Money内）
   - 現場完了時に計算
   - 各職人の内訳表示
   - 過去の計算履歴

## 運用ルール

### 評価タイミング
- **定期評価**: 3ヶ月ごと
- **随時評価**: 新しいスキル達成時

### 報酬計算タイミング
- **現場完了時**: 利益確定後に計算
- **月次精算**: 複数現場をまとめて計算も可

### パラメータ調整
- **半年ごとに見直し**
- 報酬分布・納得度を可視化して確認
- p値の調整で格差をコントロール
